---
title: 'Get Current User'
api: 'GET /api/v1/users/me'
description: 'Get the authenticated user profile information'
---

## Authentication

This endpoint requires a valid JWT access token in the Authorization header.

## Response

<ResponseField name="id" type="uuid" required>
  Unique identifier of the user
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="full_name" type="string" required>
  User's full name
</ResponseField>

<ResponseField name="phone" type="string | null">
  User's phone number (optional)
</ResponseField>

<ResponseField name="is_active" type="boolean" required>
  Whether the user account is active
</ResponseField>

<ResponseField name="is_validated" type="boolean" required>
  Whether the user email has been validated
</ResponseField>

<ResponseField name="role_name" type="string | null">
  Name of the user's role (e.g., "admin", "customer")
</ResponseField>

<ResponseField name="business_name" type="string | null">
  User's business name (optional)
</ResponseField>

<ResponseField name="occupation" type="string | null">
  User's occupation (optional)
</ResponseField>

<ResponseField name="created_at" type="datetime" required>
  Timestamp when the user was created
</ResponseField>

<ResponseField name="updated_at" type="datetime" required>
  Timestamp when the user was last updated
</ResponseField>

## Example Request

```bash
curl -X GET https://api.calzadojr.com/api/v1/users/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Status Codes

<ResponseField name="200" type="Success">
  User profile retrieved successfully
</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing or invalid authentication token
</ResponseField>

<ResponseField name="403" type="Forbidden">
  User account is not active or validated
</ResponseField>
