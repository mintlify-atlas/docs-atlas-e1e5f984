---
title: 'Forgot Password'
api: 'POST /api/v1/auth/forgot-password'
---

Requests a password recovery email. If the email is registered in the system, an email will be sent with a password reset link containing a recovery token.

## Request Body

<ParamField body="email" type="string" required>
  Email address of the account for which to request password recovery.
</ParamField>

## Response

<ResponseField name="message" type="string">
  Generic success message. The same message is returned regardless of whether the email exists to prevent email enumeration attacks.
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST https://api.calzadojyr.com/api/v1/auth/forgot-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "juan.perez@example.com"
  }'
```

</RequestExample>

<ResponseExample>

```json 200 OK
{
  "message": "Si el email está registrado, recibirás un enlace de recuperación"
}
```

</ResponseExample>

## Notes

- The response is always the same regardless of whether the email exists in the system. This prevents attackers from using this endpoint to discover valid email addresses.
- The password reset token sent via email has a limited validity period (typically 1-24 hours).
- The token can only be used once to reset the password.
